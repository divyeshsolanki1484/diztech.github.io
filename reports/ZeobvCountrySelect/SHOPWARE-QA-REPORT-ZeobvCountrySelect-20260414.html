Specific findings worth attention

1. Credential exposure (warning)

Resources/scripts/storefront-zeobv-store-api/token.twig still returns salesChannelContext.salesChannel.accessKey:

{% set response = services.response.json({
    'key': hook.salesChannelContext.salesChannel.accessKey,
}) %}

The access key is the public store-api key (not a secret). Flagged because:

  • CHANGELOG 3.0.2 says "Removed store-api credentials from twig" but the file still exists.
  • CHANGELOG 4.0.2 says "Removed context token from Twig script response… preventing token leakage", yet header.html.twig:14 still embeds data-zeobv-country-select-context-token="{{ app.session.get('sw-context-token') }}" in HTML. The context token is session-bound and should not be in HTML.

2. innerHTML interpolation (warning)

country-select.plugin.js:43:

return `<li class="zeobv-country-select__option" data-value="${element.id}">${element.name}</li>`

Country names are currently safe ISO names but the template-literal-into-innerHTML pattern is a latent XSS trap. Switch to textContent or DOM construction.
